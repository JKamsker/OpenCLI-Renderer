Automate your CLI docs

Generate InSpectraUI documentation in CI and deploy to GitHub Pages, attach as a release asset, or download as an artifact. One workflow call, zero config.

Getting Started
Add one step to your workflow

Install your CLI, then call the InSpectra action. It handles .NET, InSpectra, rendering, and verification. XML documentation is auto-detected and used for enrichment when available.

# .github/workflows/docs.yml steps: - uses: actions/checkout@v6 - run: dotnet tool install -g MyCli.Tool - uses: JKamsker/InSpectra@v1 with: cli-name: mycli output-dir: docs/cli - uses: actions/upload-artifact@v4 with: name: cli-docs path: docs/cli
# Render from pre-exported opencli.json steps: - uses: actions/checkout@v6 - uses: JKamsker/InSpectra@v1 with: mode: file opencli-json: docs/opencli.json xmldoc: docs/xmldoc.xml # optional output-dir: docs/cli
# Generate Markdown instead of HTML steps: - uses: actions/checkout@v6 - run: dotnet tool install -g MyCli.Tool - uses: JKamsker/InSpectra@v1 with: cli-name: mycli format: markdown # or markdown-monolith output-dir: docs/cli
# Build your CLI from source, then render steps: - uses: actions/checkout@v6 - uses: actions/setup-dotnet@v5 with: dotnet-version: 10.0.x - run: dotnet build src/MyCli --configuration Release - run: dotnet publish src/MyCli -o ./publish --no-build -c Release - uses: JKamsker/InSpectra@v1 with: cli-name: ./publish/mycli # path to built binary output-dir: _site
# Attach docs to a GitHub Release steps: - uses: actions/checkout@v6 - run: dotnet tool install -g MyCli.Tool - uses: JKamsker/InSpectra@v1 with: cli-name: mycli output-dir: cli-docs - run: zip -r cli-docs.zip cli-docs/ - uses: softprops/action-gh-release@v2 with: files: cli-docs.zip

The action installs .NET and InSpectra.Gen automatically. It does not install your CLI — add that step before the action.

Reference
Inputs

All inputs accepted by JKamsker/InSpectra@v1.

Automatic XML enrichment

In exec mode, the action automatically probes for cli xmldoc support and uses it when available. No flag needed — richer descriptions are included transparently.

Inputs
mode
Render mode: exec invokes a live CLI, file reads from saved JSON.
Default: exec
format
Output format: html (interactive SPA), markdown (tree), or markdown-monolith (single file).
Default: html
cli-name
CLI executable name or path. Required for exec mode.
Required (exec mode)
opencli-json
Path to your opencli.json file. Required for file mode.
Required (file mode)
xmldoc
Path to xmldoc.xml for enrichment. File mode only.
Optional
output-dir
Directory where the output is written.
Default: inspectra-output
opencli-args
Override the OpenCLI export arguments (exec mode).
Default: cli opencli
xmldoc-args
Override the xmldoc export arguments (exec mode).
Default: cli xmldoc
timeout
Timeout in seconds for each CLI export command (exec mode).
Optional
extra-args
Additional flags forwarded to the inspectra CLI.
Optional
inspectra-version
Pin a specific InSpectra.Gen NuGet version.
Default: latest
dotnet-version
.NET SDK version to install.
Default: 10.0.x
dotnet-quality
.NET SDK quality channel (preview for pre-release SDKs).

Deployment
GitHub Pages

Add a deploy job after the generate job. The reusable workflow uploads the artifact; you control when and how it gets deployed.

1. Configure your repo — Go to Settings → Pages and set the source to GitHub Actions (not "Deploy from a branch").

2. Add a deploy job — Download the artifact, re-upload as a Pages artifact, and deploy. Grant pages: write and id-token: write permissions.

.github/workflows/docs.yml
name: Deploy CLI Docs on: push: branches: [main] permissions: contents: read pages: write id-token: write jobs: generate: runs-on: ubuntu-latest steps: - uses: actions/checkout@v6 - run: dotnet tool install -g MyCli.Tool - uses: JKamsker/InSpectra@v1 with: cli-name: mycli output-dir: _site - uses: actions/upload-pages-artifact@v3 with: { path: _site } deploy: needs: generate runs-on: ubuntu-latest environment: name: github-pages url: ${{ steps.deploy.outputs.page_url }} steps: - uses: actions/deploy-pages@v4 id: deploy
Custom domain

To use a custom domain, configure it in your repository's Pages settings. GitHub handles the CNAME automatically when deploying via Actions.

Requirements
Prerequisites

Your CLI must support the OpenCLI specification for InSpectra to generate documentation from it.

For exec mode, your CLI needs to implement the cli opencli command which outputs the OpenCLI JSON spec to stdout. Optionally implement cli xmldoc for richer descriptions. CLIs built with Spectre.Console.Cli get this for free.

For file mode, export your opencli.json once and check it into your repository. This works with any CLI that can produce an OpenCLI spec, even if it's not available at CI runtime.

If your CLI uses custom export arguments (not cli opencli), pass them via opencli-args. Similarly, use xmldoc-args to override the XML documentation export command.

Custom export arguments
# If your CLI uses different export commands - uses: JKamsker/InSpectra@v1 with: cli-name: mycli opencli-args: 'export spec' # instead of 'cli opencli' xmldoc-args: 'export xmldoc' # instead of 'cli xmldoc' output-dir: docs
