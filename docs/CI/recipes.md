## Deploy to GitHub Pages

Generates an HTML bundle on every push to `main` and serves it from your
project's GitHub Pages site.

**One-time setup:** in your repository, go to **Settings → Pages** and set
the source to **GitHub Actions** (not "Deploy from a branch").

```yaml
# .github/workflows/docs.yml
name: Deploy CLI Docs

on:
  push:
    branches: [main]
  workflow_dispatch:

permissions:
  contents: read
  pages: write
  id-token: write

concurrency:
  group: pages
  cancel-in-progress: true

jobs:
  generate:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: JKamsker/InSpectra@v1
        with:
          mode: dotnet
          project: src/MyCli
          configuration: Release
          output-dir: _site

      - uses: actions/upload-pages-artifact@v3
        with:
          path: _site

  deploy:
    needs: generate
    runs-on: ubuntu-latest
    environment:
      name: github-pages
      url: ${{ steps.deploy.outputs.page_url }}
    steps:
      - uses: actions/deploy-pages@v4
        id: deploy
```

> **Custom domain?** Configure it in your repository's Pages settings —
> GitHub handles the CNAME automatically when deploying via Actions.

## Open a PR with regenerated Markdown

The pattern for "always-fresh CLI reference checked into the repo": render
Markdown into a folder, then let
[`peter-evans/create-pull-request`](https://github.com/peter-evans/create-pull-request)
open or update a PR.

```yaml
# .github/workflows/update-cli-docs.yml
name: Update CLI Docs

on:
  push:
    branches: [main]
    paths:
      - 'src/MyCli/**'
      - '.github/workflows/update-cli-docs.yml'
  workflow_dispatch:

permissions:
  contents: write
  pull-requests: write

jobs:
  regenerate:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: JKamsker/InSpectra@v1
        with:
          mode: dotnet
          project: src/MyCli
          format: markdown
          output-dir: docs/cli

      - uses: peter-evans/create-pull-request@v6
        with:
          branch: chore/update-cli-docs
          title: 'docs: regenerate CLI reference'
          commit-message: 'docs: regenerate CLI reference from src/MyCli'
          body: |
            Automated update of `docs/cli/` from `src/MyCli`
            at commit ${{ github.sha }}, generated by
            the generate-then-render `dotnet` flow via JKamsker/InSpectra@v1.
          add-paths: docs/cli
          delete-branch: true
          labels: docs, automated
```

**Why each piece is here**

- **Path filter** scopes the trigger to changes that could actually affect
  the CLI surface, so unrelated commits don't churn PRs.
- **`add-paths: docs/cli`** keeps any incidental files out of the commit
  (the auto-installed `InSpectra.Cli` package mutates your `.csproj` at
  runtime — `add-paths` prevents that mutation from leaking into the PR).
- **`delete-branch: true`** cleans up the working branch on merge.

## Drift check on pull requests

Pair the auto-PR job above with a drift check that fails PRs whose `docs/cli`
is out of date — so contributors regenerate before merging instead of
waiting for the post-merge auto-PR.

```yaml
# .github/workflows/check-cli-docs.yml
name: Check CLI Docs

on:
  pull_request:
    paths:
      - 'src/MyCli/**'
      - 'docs/cli/**'

permissions:
  contents: read

jobs:
  drift:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: JKamsker/InSpectra@v1
        with:
          mode: dotnet
          project: src/MyCli
          format: markdown
          output-dir: /tmp/cli-docs-fresh

      - name: Compare with checked-in docs
        run: |
          if ! diff -r --brief docs/cli /tmp/cli-docs-fresh; then
            echo "::error::docs/cli is out of date — regenerate locally and commit."
            exit 1
          fi
```

## Attach docs to a GitHub Release

```yaml
# .github/workflows/release-docs.yml
name: Release Docs

on:
  release:
    types: [published]

jobs:
  attach:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: JKamsker/InSpectra@v1
        with:
          mode: dotnet
          project: src/MyCli
          configuration: Release
          label: ${{ github.ref_name }}    # e.g. v1.2.3 — shown in the viewer header
          output-dir: cli-docs

      - run: zip -r cli-docs-${{ github.ref_name }}.zip cli-docs/

      - uses: softprops/action-gh-release@v2
        with:
          files: cli-docs-${{ github.ref_name }}.zip
```
